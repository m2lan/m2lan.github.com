发表于 2018-09-17   |   分类于 Thinkphp5   |  

创建think5自定义命令行

ThinkPHP5.0支持Console应用，通过命令行的方式执行一些URL访问不方便或者安全性较高的操作

创建命令行类

  • 创建application/console/WebSocket.php

    namespace app\console;

use think\console\Command;
use think\console\Input;
use think\console\Output;

class WebSocket extends Command {

    // 命令行配置函数
    protected function configure() {
        // setName: 设置命令行名称
        // setDescription: 设置命令行描述
        $this->setName('websocket:start')->setDescription('启动WebSocket服务器.');
    }

    // 设置命令行返回信息
    protected function execute(Input $input, Output $output) {
        $output->writeln("WebSocket: 启动.\n");
    }
}

清除缓存的域名

  • Windows下使用ipconfig /flushdns
  • Linux下使用/etc/init.d/nscd restart

域名解析方式

  • A记录
    • A代表的是Address，用来指定域名对应的IP地址。A记录可以将多个域名解析到一个IP地址。
  • MX记录
    • 表示Mail Exchange，可以将某个域名下的邮件服务器指向自己的Mail Server，如m2lan.com域名的A记录IP地址是39.108.217.213，如果将MX记录设置为39.108.217.213，即xxx@m2lan.com的邮件路由，DNS会将邮件发送到39.108.217.213所在到服务器，而正常通过WEB请求到话仍然解析到A记录到IP地址
  • CNAME记录
    • 别名解析，可以为一个域名设置一个或多个别名
  • NS记录
    • 为某个域名指定DNS解析服务器，也就是这个域名有指定的IP地址的DNS服务器去解析
  • TXT记录
    • 为某个主机名或域名设置说明，如可以为m2lan.com设置TXT记录为hello,world这样的说明

HTTP Headers

发表于 2017-12-06   |   分类于 HTTP   |  

HTTP Headers

常见的HTTP请求头

请求头 说明
Accept-Charset 用于指定客户端接受的字符集
Accept-Encoding 用于指定可接受的内容编码，如Accept-Encoding:gzip.deflate
Accept-Language 用于指定一种自然语言，如Accept-Language:zh-cn
Host 用于指定被请求资源的Internet主机和端口号，如Host:www.m2lan.com
User-Agent 客户端将它的操作系统、浏览器和其他属性告诉服务器
Connection 当前连接是否保持，如Connection:Keep-Alive

常见的HTTP响应头

请求头 说明
Server 使用的服务器名称，如Server:nginx/1.10.2
Content-Type 用来指明发送给接收者的实体正文的媒体类型，如Content-Type:text/html; charset=utf-8
Content-Encoding 与请求报头Accept-Encoding对应，告诉浏览器服务端采用的是什么压缩编码
Content-Language 描述了资源所用的自然语言，与Accept-Language对应
Content-Length 指明实体正文的长度，用以字节方式存储的十进制数字来表示
Keep-Alive 保持连接的时间，如Keep-Alive: timeout=5,max=120

常见的HTTP状态码

请求头 说明
200 客户端请求成功
302 临时跳转，跳转的地址通过Location指定
400 客户端请求有语法错误，不能被服务器识别
403 服务器收到请求，但是拒绝提供服务
404 请求的资源不存在
500 服务器发生不可预期的错误

浏览器缓存机制

  • Cache-Control/Pragma HTTP Head字段的可选值
可选值 说明
Public 所有内容都将被缓存，在响应头中设置
Private 内容只缓存到私有缓存中，在响应头中设置
no-cache 所有内容都不会被缓存，在请求头和响应头中设置
no-store 所有内容都不会被缓存到缓存或Internet临时文件中，在响应头中设置
must-revalidation/proxy-revalidation 如果缓存的内容失效，请求必须发送到服务器/代理以进行重新验证，在请求头中设置
max-age=xxx 缓存的内容将在xxx秒后失效，这个选项只在HTTP1.1中可用，和Last-Modified一起使用时优先级较高，在响应头中设置

  • Expires

    • 使用格式是Expires:Thu, 07 Dec 2017 08:23:13 GMT，超过这个时间值后，缓存的内容将失效。

  • Last-Modified/Etag

    • 表示一个服务器上的资源的最后修改时间
    • 资源可以是静态(静态内容自动加上Last-Modified字段)或者动态(如servlet提供了一个getLastModified方法用于检查某个动态内容是否已经更新)，通过这个最后修改时间可以判断当前请求的资源是否是最新的
    • 一般服务器在响应头中返回一个Last-Modified字段，告诉浏览器这个页面的最后修改时间Last-Modified:Thu, 07 Dec 2017 08:31:01 GMT，浏览器再次请求时在请求头中增加一个If-Modified-Since:Thu, 07 Dec 2017 08:31:01 GMT字段，询问当前缓存的页面是否是最新的，如果是最新的就返回304状态码，告诉浏览器是最新的，服务器也不会传输新的书籍
    • 与Last-Modified字段有类似功能的Etag字段，这个字段的作用是让服务器给每个页面分配一个唯一的编号，然后通过这个编号来区分当前这个页面是否是最新的

Spring自定义组件标签

spring自定义组件标签步骤

  • 采用XSD描述自定义标签的元素属性
  • 编写Bean定义的解析器
  • 注册自定义标签解析器
  • 绑定命名空间解析器

定义标签元素的XML结构，采用XSD描述自定义标签的元素属性

  • 在com.example.spring03.config包下创建userservice.xsd

    <?xml version="1.0" encoding="UTF-8"?>
    <xsd:schema xmlns="http://www.m2lan.com/schema/service"
                xmlns:xsd="http://www.w3.org/2001/XMLSchema"
                xmlns:beans="http://www.springframework.org/schema/beans"
                <!-- 指定一个自定义标签的命名空间 -->
                targetNamespace="http://www.m2lan.com/schema/service"
                elementFormDefault="qualified"
                attributeFormDefault="unqualified">

        <!-- 导入Spring本身的beans命名空间 -->
        <xsd:import namespace="http://www.springframework.org/schema/beans"/>

        <!-- 定义一个user-service标签 -->
        <xsd:element name="user-service">
            <xsd:complexType>
                <xsd:complexContent>
                    <xsd:extension base="beans:identifiedType">

                        <!-- 扩展属性dao -->
                        <xsd:attribute name="dao" type="xsd:string" use="required"/>
                    </xsd:extension>
                </xsd:complexContent>
            </xsd:complexType>
        </xsd:element>

    </xsd:schema>
